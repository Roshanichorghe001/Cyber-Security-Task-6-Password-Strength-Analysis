# Cyber-Security-Task-6-Password-Strength-Analysis
Evaluated password strength using an online password checker, analyzed weak to very strong passwords, and documented password security best practices.

## Objective
The objective of this task was to understand the importance of strong passwords by creating passwords of different strengths and evaluating them using an online password strength checker.

## Tool Used
- Password Strength Checker (https://passwordmeter.com/)

---

## Password Evaluation Results

### 1. Password: 123456789
**Strength:** Very Weak

**Observations:**
- One of the most commonly used passwords.
- Extremely easy to guess.
- Can be cracked instantly.
- Does not provide any security.

---

### 2. Password: Dog123
**Strength:** Very Weak

**Observations:**
- Too short.
- Contains a common keyboard pattern ("123").
- Easy to crack using dictionary or brute-force attacks.

---

### 3. Password: Shikshya#123
**Strength:** Fair

**Observations:**
- Good password length.
- Uses uppercase/lowercase letters, numbers, and special characters.
- Contains the predictable pattern "123", making it less secure.

---

### 4. Password: Shiksa#T568?U9
**Strength:** Strong

**Observations:**
- Good password length.
- Includes uppercase letters, lowercase letters, numbers, and special characters.
- No common patterns detected.
- Highly resistant to brute-force attacks.

---

### 5. Password: Shiksyhua#T568?U9
**Strength:** Very Strong

**Observations:**
- Excellent password length.
- Uses all four character types.
- No common or predictable patterns.
- Extremely difficult to crack.
- Provides excellent protection against brute-force and dictionary attacks.

---

## Key Learnings

- Password length plays a significant role in password security.
- Avoid using common passwords like "123456789".
- Do not use predictable patterns such as "123" or personal information.
- Use uppercase letters, lowercase letters, numbers, and special characters.
- Create unique passwords for different accounts.
- Enable Multi-Factor Authentication (MFA) for additional security.

---

## Conclusion

This task helped me understand how password complexity directly affects security. I learned how weak passwords can be cracked quickly, while strong and unique passwords significantly improve protection against brute-force and dictionary attacks. This exercise also emphasized the importance of using long, unpredictable passwords and enabling Multi-Factor Authentication for better account security.
